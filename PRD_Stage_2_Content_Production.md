# PRD Stage 2: 콘텐츠 제작 및 리퍼퍼징 엔진

## 1. 목적
Stage 1에서 선별된 소재를 바탕으로 플랫폼별 최적화된 콘텐츠를 자동 생성하여 "One Source Multi Use"를 극대화함.

## 2. 주요 기능 및 사용자 시나리오
### F1. 플랫폼별 맞춤형 글쓰기 (Copywriting)
*   **동작**: 하나의 주제를 입력하면 AI가 4가지 버전의 원고를 작성함.
    *   **블로그**: 소제목이 있는 장문의 정보성 글.
    *   **인스타그램**: 감성적인 말투와 이모지가 섞인 짧은 글.
    *   **쓰레드**: 질문과 답변 형식의 소통형 글.
    *   **트위터**: 팩트 중심의 짧은 정보 스레드.

### F2. 비주얼 에셋 자동 생성
*   **동작**: 원고 내용에 맞는 이미지 생성을 위한 프롬프트를 AI가 생성하고, 이를 Gemini Image API (`gemini-2.5-flash-image`)에 전달하여 이미지 생성.
*   **결과물**: 카드뉴스 배경, 블로그 썸네일, 트위터 삽입용 이미지.

### F3. 콘텐츠 패키징 (Packaging)
*   **동작**: 생성된 텍스트와 이미지를 하나의 폴더 또는 데이터베이스 행에 결합하여 저장.
*   **검수 기능**: 슬랙(Slack)으로 관리자에게 "승인하시겠습니까?" 메시지 발송.

## 3. 기술 스택
*   **AI (Text)**: Gemini 2.5 Pro (`gemini-2.5-pro` 모델, `@google/genai` 라이브러리 연동)
*   **AI (Image)**: Gemini Image API (`gemini-2.5-flash-image` 모델)
*   **API Auth**: `GEMINI_API_KEY` 환경변수 기반 인증
*   **Orchestration**: n8n (Code Node 활용하여 플랫폼별 말투 템플릿 관리)

## 4. 성공 지표 (KPI)
*   소재 입력 후 전체 채널용 콘텐츠 생성 시간 10분 이내.
*   AI 생성 콘텐츠의 수정 비율 20% 미만 유지.
